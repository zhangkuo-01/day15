1、activeMQ：是一个与平台无关的消息中间件，主要用于应用解耦。
2、有两个角色：消息的生成者、消息的消费者
3、有5种消息类型：
· TextMessage--一个字符串对象 json （是我们项目中最常用的！！！）
· MapMessage--一套名称-值对
· ObjectMessage--一个序列化的 Java 对象
· BytesMessage--一个字节的数据流
· StreamMessage -- Java 原始值的数据流

4、有两种消息模式： （重点掌握！！！！！）
点对点模式  Queue
点对点特点： 送快递
1）一个消息生产者对应一个消息消费者
2）消息只能被一个消费消费，且只消费一次
3）如果消息生产完，消费者不在线，消息进入队列并持久化保存，当消费者上线后自动消费



发布订阅模式： topic 教室广播
特点：
1）一个消息生产者对应多个消息消费者
2）消息可以被多个消费者消费，但是每个消费者仅能消费一次
3）如果消息生产完，消费者不在线，则消息丢失，默认不会持久化保存


队列类型不一样

 * 注意！！！！
 * 消费者接收消息时候：
 * 1、监听的消息队列的类型与队列的名称必须跟生产者发送的消息队列类型及名称保持一致;
 * 否则就会出现接受不到消息
 * 
 * 2、接收消息时候，消息类型必须与发送消息的类型保持一致；
 * 否则会出现类型转换异常
